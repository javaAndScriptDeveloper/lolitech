# Big Data — Lab Work

An end-to-end distributed data processing pipeline: synthetic employee salary records are ingested into HDFS, aggregated and currency-converted with a PySpark job, and the results are exposed as a queryable Hive table.

**Stack:** Python · PySpark · Apache Spark 3.0 · Hadoop 3.2 (HDFS + YARN) · Hive 2.3 · PostgreSQL · Docker Compose · Bash

---

## Architecture

```
employee_data.csv  ─┐
                    ├─▶  HDFS (/data/)  ─▶  Spark job  ─▶  HDFS result.csv  ─▶  Hive table
currency_rates.json─┘
```

The full cluster runs locally via Docker Compose and consists of:

| Service | Role |
|---------|------|
| **namenode / datanode** | HDFS distributed file system |
| **resourcemanager / nodemanager** | YARN resource allocation |
| **spark-master / spark-worker** | Spark standalone cluster |
| **hive-server / hive-metastore** | HiveServer2 + Thrift metastore |
| **hive-metastore-postgresql** | PostgreSQL backing the Hive metastore |
| **historyserver** | MapReduce job history |
| **presto-coordinator** | Optional federated query layer |

## Data

`data/employee_data.csv` — 1 million synthetic rows generated by `generate_employee_csv.py`:

```
employee_id,salary,currency
42,850,USD
17,320,EUR
...
```

Employee IDs range 0–1000 (so each employee has many salary records). Currencies are USD, EUR, or CAD. `currency_rates.json` holds the conversion rates applied during aggregation.

---

## Spark Job ([`spark_job.py`](./spark_job.py))

Runs as a PySpark application submitted to the Spark standalone cluster:

1. **Load** — reads the CSV from HDFS and the currency rates JSON.
2. **Broadcast** — distributes the currency rates dict to all worker nodes as a broadcast variable, avoiding a shuffle when looking up rates per employee.
3. **First MapReduce pass** — maps each row to `(employee_id, (salary, currency))`, groups by employee ID, and accumulates a running sum and count.
4. **Second pass** — for each employee, divides total salary by count to get the average, then multiplies by the employee's conversion rate.
5. **Write** — saves the result (`employee_id`, `avg_salary_converted`) as CSV back to HDFS.

---

## Hive Table ([`data/create_employee_table.sql`](./data/create_employee_table.sql))

After the Spark job completes, an external Hive table is created on top of the output CSV:

```sql
CREATE EXTERNAL TABLE employee_salary (
  employee_id INT,
  avg_salary  FLOAT
)
ROW FORMAT DELIMITED FIELDS TERMINATED BY ','
STORED AS TEXTFILE
LOCATION 'hdfs://namenode:9000/data/result.csv';
```

Results are queried through Beeline (HiveServer2 JDBC) to validate row counts.

## Running

```bash
# Start the cluster, upload data, run Spark job, create Hive table, query result
bash run.sh
```

`run.sh` handles the full sequence: `docker-compose up` → HDFS safe-mode off → upload data → spark-submit → Hive DDL → Beeline validation.

Web UIs once running:

|----|-----|
